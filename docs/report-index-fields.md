# Report Index Fields

`reports\dbyte-agent-index.json` is a read-only inventory of local DBYTE agent and pool-core report files.

## Top-level fields

| Field | Meaning |
| --- | --- |
| `index_schema` | Report index schema version. |
| `index_scope` | Must be `read_only`. |
| `index_ts_unix` | Unix timestamp when the index was generated. |
| `index_status` | `ok` when all required reports are present, otherwise `attention`. |
| `report_count` | Number of report entries tracked by the index. |
| `missing_required_count` | Number of required report entries missing from disk. |
| `reports` | Array of report entries. |

## Report entry fields

| Field | Meaning |
| --- | --- |
| `name` | Stable report label for operator display. |
| `kind` | Report format, such as `json` or `text`. |
| `path` | Local report path relative to the repository root. |
| `required` | `true` means the full agent gate expects the file to exist. |
| `exists` | Whether the file was present when the index was generated. |
| `status` | `present` or `missing`. |
| `sha256` | SHA256 hash when present, otherwise `<missing>`. |
| `size_bytes` | File size in bytes when present, otherwise `0`. |

## Required pool-core entry

The index includes `pool_core_ledger` for `reports\dbyte-pool-ledger-report.json`.

This entry is generated by the pool-core verification gate before the agent report index is exported. The full gate requires it to be present so dashboard and operator tooling can discover the pool-core replay summary through the same read-only index surface as the agent artifacts.

When present, `pool_core_ledger` also embeds replay summary fields from the pool-core report JSON:

| Field | Meaning |
| --- | --- |
| `replay_schema` | Pool-core replay report schema version. |
| `replay_status` | Pool-core replay report status. |
| `replay_total_events` | Total replay event count. |
| `replay_accepted_events` | Accepted replay event count. |
| `replay_rejected_events` | Rejected replay event count. |
| `replay_credited_difficulty` | Credited difficulty total. |
| `replay_session_count` | Number of replay session rows. |

Simple rule: if `missing_required_count` is not `0`, the operator should regenerate reports with `npm run verify` before trusting dashboard status.
